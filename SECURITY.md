# Security Policy

## Supported Versions

Only the latest `main` branch is supported for security updates.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

**Please do not open a public GitHub issue for security vulnerabilities.**

Reporting a vulnerability is the first step towards making RustRay more secure. We take all security reports seriously.

### How to report
Please send your vulnerability report via email to: **security@edgeray.io**.

We aim to acknowledge your report within 48 hours and provide a fix within 14 days for major vulnerabilities.

### What to include
- A clear description of the vulnerability.
- Steps to reproduce (Proof of Concept).
- Potential impact and severity.
- Recommended fix if possible.

## Ethical Disclosure
We appreciate everyone who follows [Responsible Disclosure](https://en.wikipedia.org/wiki/Responsible_disclosure) to help keep users safe.
- Do not perform destructive actions on public infrastructure.
- Do not attempt to access user data.
- Give us reasonable time to fix the issue before public disclosure.

Thank you for helping keep the EdgeRay community safe! 🛡️
